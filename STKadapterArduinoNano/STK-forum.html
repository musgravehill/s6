STK alternative using Arduino Nano

Here is the sketch that allows turning a ~2 cheap, 16MHz/5V Arduino Nano 328P or Arduino Nano 168 into the STK module.

Arduino Nano as STK Dongle (1 min 3 sec)


It has two modes:
1) S6R config mode (default) that can be used for configuration of the S6R receiver using the FrSky's S6RConfig Tool or Mike's SportSet
2) Firmware upgrade/FrSky sensor config mode (requires jumper on two of the ISP connector pins when powering up - see attached Connections Diagram). For firmware upgrade use FrSky's S.Port Upgrade Tool. For FrSky sensors configuration use Frsky's SportTool (Vario/FLVSS/FCS) or Mike's SportSet (SP2UART/RPM). One additional feature in this mode is a built in sensor hardware ID changing tool. In order to enter the ID changing mode you need copy&paste ~~~ (3 tildas) sequence (hexadecimal 7E7E7E) into your serial terminal (connected at 57600 baud), and follow the on screen instructions. Here is an example session:

Code:
=========================
 FrSky Sensor ID changer
=========================

Looking for sensor...
Found sensor with ID = 2

Press +/- to select new sensor ID, S to set the new ID,
R to restore sensor to factory defaults or X to exit without changes
New sensor ID = 2

Press +/- to select new sensor ID, S to set the new ID,
R to restore sensor to factory defaults or X to exit without changes
New sensor ID = 3
Changing sensor ID from 2 to 3...
Looking for sensor...
Found sensor with ID = 3
Sensor ID changed succesfully.

Exited.
To connect it you need to swap the S.Port signal and 5V wire in the S.Port cable (see attached Connections Diagram). S6R is powered from the Arduino Nano so don't connect any external power to it.

The attached hex file can be uploaded to your Arduino Nano using XLoader. Select the COM port your Arduino Nano uses and set the configuration as on attached XLoader Configuration 328P or XLoader Configuration 168 picture (depending on the processor you use).
